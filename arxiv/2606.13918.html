О чём статья

Авторы представляют слой байесовской калибровки для детекторов «slopsquat» - тех, что помечают галлюцинированные импорты пакетов в коде, сгенерированном большими языковыми моделями (LLM). Там, где существующие конвейеры выдают бинарные решения (пометить / не помечать), предложенный слой выдаёт апостериорную вероятность по бета-распределению для каждой находки. Она выводится из эпистемической таксономии трёх категорий, которая явно классифицирует каждое априорное предположение как эмпирически калиброванное, конструктивно обоснованное или основанное на инженерном суждении.

Помимо основного канала реестра (ответы 200/404), калиброванный слой использует сигналы метаданных PyPI - возраст пакета, число релизов, описание автора, сводку - чтобы выявлять зарегистрированные, но подозрительные пакеты, которые бинарный детектор реестра пропускает. Это и есть реалистичный режим атаки уже после генерации кода LLM. Полученный примитив, учитывающий риск, напрямую пригоден для CI-гейтов и поддерживает осмысленный выбор порогов для разных правил детекции.

Калибровка оценивается на объединённом корпусе из 1 734 Python-фрагментов: стратифицированный срез из 189 промптов BigCodeBench плюс стресс-набор из 100 промптов по нишевым библиотекам, сгенерированных панелью из шести моделей (четыре облачные - Claude-Sonnet-4.6, Mistral-Large, DeepSeek-v4-pro, DeepSeek-R1 - и две локальные открытые кодовые модели Mistral Codestral и Meta CodeLlama). Асимметрию детектора авторы оценивают парным тестом Макнемара, а калибровку - как через Expected Calibration Error на помеченном подмножестве, так и через строго корректную оценку Брайера по полному корпусу.

Экспертное мнение

Галлюцинация импорта - это не мелкая ошибка, а потенциальная supply-chain атака.
Когда модель придумывает пакет, разработчик может не заметить подмену или подтянуть подозрительную зависимость. Veai работает из JetBrains IDE и видит реальные зависимости проекта, индексы, исходники и ошибки резолва, поэтому помогает отличать существующий API от выдуманного имени ещё до того, как такой код попадёт в репозиторий.
